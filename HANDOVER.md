# 🌞 Sunstore — Документация по передаче проекта

## 📋 Обзор проекта

**Sunstore** — полнофункциональный интернет-магазин солнечных панелей и энергосистем с российской платёжной инфраструктурой.

### ✅ Что реализовано

#### 1. Витрина магазина (клиентская часть)
- **Главная страница**: Apple-style дизайн с минималистичным hero-секцией, каталогом товаров, блоком преимуществ "Почему Sunstore"
- **Каталог товаров**: Сетка из 8+ solar-продуктов (панели, инверторы, аккумуляторы, контроллеры, крепёжные системы)
- **Детальные страницы товаров**: Array, Blueprint, Velocity, Status — с фильтрами, ценами и кнопками добавления в корзину
- **Корзина**: Редактирование количества, расчёт стоимости доставки (бесплатно от ₽50,000), переход к оформлению
- **Оформление заказа**: Сбор данных клиента → выбор способа оплаты → оплата через шлюз

#### 2. Платёжная система (6 шлюзов)
Всё готово для приёма платежей на территории РФ:

| Шлюз | Режим песочницы | Боевой режим | Поддерживаемые методы |
|------|----------------|-------------|---------------------|
| **Т-Банк (Тинькофф)** | `https://rest-api-test.tinkoff.ru/v2/` | `https://securepay.tinkoff.ru/v2/` | Карты РФ, СБП, T-Pay |
| **ЮKassa (Яндекс)** | Тестовый shopId/secretKey | Live shopId/secretKey | Карты РФ, СБП, SberPay, Кошельки |
| **Сбербанк (SberPay)** | `https://3dsec.sberbank.ru/payment/rest` | `https://securepayments.sberbank.ru/payment/rest` | Карты РФ, SberPay, СБП |
| **CloudPayments** | Тестовый Public ID + API-пароль | Live Public ID + API-пароль | Карты, СБП, виджет оплаты |
| **Робокасса** | IsTest=1 | Без флага | Карты, СБП, Кошельки |
| **Демо-режим** ✅ | Активен по умолчанию | N/A | Имитация успешной/отклонённой оплаты |

**По умолчанию активен Демо-режим** — магазин работает сразу без merchant-ключей. Переключение на реальные шлюзы — в админке (Настройки → Платежи).

#### 3. Админ-панель (/admin)
Полноценная система управления магазином:

- **Обзор (Dashboard)**: KPI — выручка, количество заказов, средний чек, товары с низким остатком
- **Управление товарами**: Создание/редактирование/удаление товаров, управление ценами и складом
- **Заказы**: Просмотр всех заказов, изменение статуса (Ожидает → Оплачен → Отправлен → Доставлен / Отменён)
- **Настройки платежей**: Конфигурация каждого шлюза (sandbox/live mode, учётные данные), тестирование подключения

## 💳 Как протестировать платёжный шлюз

### Вариант 1: Демо-режим (уже включён)
1. Добавьте товар в корзину
2. Перейдите к оформлению заказа
3. Заполните данные клиента (любые тестовые: name=test, email=test@test.com)
4. Нажмите "Создать заказ и перейти к оплате"
5. Выберите **"Демо-режим"** как способ оплаты
6. Вы будете перенаправлены на страницу `/pay/demo/[orderId]`
7. Два варианта:
   - **"Имитировать успешную оплату"** → заказ помечается как оплаченный, редирект на страницу статуса
   - **"Отклонить"** → заказ отменяется

✅ Этот метод не требует merchant-учётных данных и подходит для первичного тестирования.

### Вариант 2: Тестирование Т-Банк (sandbox)

#### Шаг 1: Получение учётных данных
1. Зарегистрируйтесь в Т-Банк Бизнес: https://www.tinkoff.ru/business/
2. Перейдите в раздел "Эквайринг" → "Разработчикам"
3. Получите:
   - **Terminal Key** (например: `TestTerminalXXXXXXXXXXX`)
   - **Secret Password** (пароль для подписи запросов)

#### Шаг 2: Настройка в админке
1. Войдите в `/admin` с паролем `sunstore`
2. Перейдите в **Настройки → Платежи**
3. Найдите карточку **"Т-Банк (Тинькофф)"**
4. Включите переключатель "Включить"
5. Установите режим **"Песочница"**
6. Введите полученные Terminal Key и Secret Password
7. Нажмите **"Проверить подключение"** → должно появиться "Подключение успешно"
8. Нажмите **"Сделать активным"** → Т-Банк станет основным способом оплаты
9. Нажмите **"Сохранить"** внизу страницы

#### Шаг 3: Тестирование платежа
1. Выйдите из админки, откройте витрину `https://sunstore.vercel.app`
2. Добавьте товар в корзину, перейдите к checkout
3. Выберите **"Т-Банк (Тинькофф)"**
4. Нажмите "Оплатить"
5. Вы будете перенаправлены на тестовую платежную форму Т-Банка
6. Используйте тестовые карты:
   - **Успех**: Карта `2200 0000 0000 0001`, срок `12/30`, CVV `123`
   - **Ошибка**: Карта `4444 4444 4444 4445`, срок `12/30`, CVV `123`

### Вариант 3: ЮKassa (test mode)

#### Шаг 1: Регистрация
1. Создайте тестовый магазин: https://yookassa.ru/dashboard/sandbox
2. Получите:
   - **shopId** (идентификатор магазина)
   - **live_secret_key** или **test_secret_key** (для sandbox используется test_...)

#### Шаг 2: Настройка
1. В админке: **Настройки → Платежи** → найдите "ЮKassa (Яндекс)"
2. Включите, выберите "Песочница", введите shopId и secretKey
3. Проверьте подключение → "Подключение успешно"
4. Сделайте активным и сохраните

#### Тестовые платежи работают аналогично Т-Банк

### Вариант 4: Сбербанк (sandbox)

#### Шаг 1: Получение доступа
1. Зарегистрируйтесь в "Сбербанк Эквайринг": https://www.sberbank.ru/ru/s_m_business/bankingservice/equaring
2. Запросите тестовые доступы для разработчика
3. Получите:
   - **userName** (логин магазина)
   - **password**

#### Шаг 2: Настройка
Аналогично другим шлюзам через админку.

## ⚙️ Переменные окружения (опционально)

Для persistence учётных данных между restarts Vercel можно настроить env vars:

```bash
# T-Bank
TBANK_TERMINAL_KEY=your_terminal_key
TBANK_SECRET=your_password
TBANK_TEST_MODE=true    # или false для live

# ЮKassa
YOOKASSA_SHOP_ID=your_shop_id
YOOKASSA_SECRET_KEY=your_secret_key

# Сбербанк
SBERBANK_USERNAME=your_username
SBERBANK_PASSWORD=your_password

# CloudPayments
CLOUDPAYMENTS_PUBLIC_ID=your_public_id
CLOUDPAYMENTS_API_PASSWORD=your_api_password

# Робокасса
ROBOKASSA_MERCHANT_LOGIN=your_login
ROBOKASSA_PASSWORD1=password_for_signing
ROBOKASSA_PASSWORD2=password_for_notifications

# Пароль админки
ADMIN_PASSWORD=your_secure_password
```

## 📊 Структура базы данных

Проект использует in-memory хранение через `globalThis` (resets at Vercel cold starts). Это означает:

✅ Подходит для: быстрых прототипов, демо, low-traffic магазинов  
⚠️ Не подходит для production с high traffic  

**Для production рекомендуется:**
1. Подключить PostgreSQL через Prisma ORM (schema уже есть в `db/schema.prisma`)
2. Или использовать MongoDB/Firebase
3. Настроить постоянный session store
